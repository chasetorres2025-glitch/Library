基于 Vue（前端）+ SpringBoot + MySQL（MyBatis）（后端）的图书管理系统，我会为你提供一份**详细、可落地**的技术实现方案，包含数据表结构（字段设计）、核心功能项以及关键技术实现要点，兼顾新手友好性和行业最佳实践。

## 一、整体技术架构
先明确整体技术栈和架构分层，确保前后端职责清晰：

| 层级         | 技术选型                                                                 |
|--------------|--------------------------------------------------------------------------|
| 前端         | Vue 3 + Element Plus（UI组件库）+ Vue Router（路由）+ Pinia（状态管理）+ Axios（请求） |
| 后端         | SpringBoot 3.x + MyBatis/MyBatis-Plus（数据访问）+ Spring Security（权限）+ Lombok（简化代码） |
| 数据库       | MySQL 8.x（关系型数据库）                                                |
| 开发/部署辅助 | Maven（依赖管理）+ Postman（接口测试）+ Navicat（数据库管理）             |

### 架构分层（后端）
```
SpringBoot 后端
├── controller（控制器）：接收前端请求，返回响应
├── service（服务层）：核心业务逻辑处理
├── mapper（数据访问层）：MyBatis 映射接口，操作数据库
├── entity（实体层）：对应数据库表的实体类
├── dto（数据传输对象）：前后端交互的参数/返回对象（避免直接暴露实体）
├── vo（视图对象）：前端展示专用对象
├── config（配置层）：MyBatis、Security 等配置
├── exception（异常处理）：全局异常捕获
└── utils（工具类）：通用工具（如日期、加密）
```

### 架构分层（前端）
```
Vue 前端
├── src/api（接口封装）：统一管理后端接口请求
├── src/components（公共组件）：复用组件（如分页、表单）
├── src/views（页面视图）：各功能模块页面
├── src/router（路由）：页面路由配置
├── src/store（Pinia）：全局状态管理
├── src/utils（工具类）：Axios 封装、通用方法
└── src/assets（静态资源）：样式、图片等
```

## 二、数据库表结构设计
图书管理系统核心表共 7 张，涵盖**用户权限、图书信息、借阅管理**三大核心场景，字段设计兼顾实用性和扩展性，注释清晰：

### 1. 用户表（sys_user）
存储系统用户（管理员、读者）信息，区分角色：
| 字段名       | 字段类型         | 注释                 | 约束/说明                          |
|--------------|------------------|----------------------|-----------------------------------|
| id           | bigint           | 主键 ID              | 自增（AUTO_INCREMENT），主键      |
| username     | varchar(50)      | 用户名               | 非空，唯一（UNIQUE）              |
| password     | varchar(100)     | 密码（加密后）       | 非空（存储 BCrypt 加密后的密码）  |
| real_name    | varchar(20)      | 真实姓名             | 非空                              |
| phone        | varchar(11)      | 手机号               | 唯一                              |
| email        | varchar(50)      | 邮箱                 | 可选                              |
| role_id      | bigint           | 角色 ID（关联角色表）| 非空，外键（关联 sys_role.id）    |
| status       | tinyint          | 状态（0-禁用，1-正常）| 非空，默认 1                      |
| create_time  | datetime         | 创建时间             | 非空，默认 CURRENT_TIMESTAMP      |
| update_time  | datetime         | 更新时间             | 自动更新（ON UPDATE CURRENT_TIMESTAMP） |

### 2. 角色表（sys_role）
存储角色（如管理员、普通读者），控制权限范围：
| 字段名       | 字段类型         | 注释                 | 约束/说明                          |
|--------------|------------------|----------------------|-----------------------------------|
| id           | bigint           | 主键 ID              | 自增，主键                        |
| role_name    | varchar(20)      | 角色名称             | 非空，唯一（如：管理员、读者）    |
| role_code    | varchar(20)      | 角色编码             | 非空，唯一（如：ADMIN、READER）   |
| description  | varchar(100)     | 角色描述             | 可选                              |
| create_time  | datetime         | 创建时间             | 非空，默认 CURRENT_TIMESTAMP      |

### 3. 图书分类表（book_category）
对图书进行分类（如计算机、文学、历史）：
| 字段名       | 字段类型         | 注释                 | 约束/说明                          |
|--------------|------------------|----------------------|-----------------------------------|
| id           | bigint           | 主键 ID              | 自增，主键                        |
| category_name| varchar(30)      | 分类名称             | 非空，唯一（如：计算机科学）      |
| parent_id    | bigint           | 父分类 ID（多级分类）| 可选，默认 0（一级分类）          |
| sort         | int              | 排序权重             | 可选，默认 0（值越小越靠前）      |
| create_time  | datetime         | 创建时间             | 非空，默认 CURRENT_TIMESTAMP      |

### 4. 图书信息表（book_info）
核心表，存储图书基本信息：
| 字段名       | 字段类型         | 注释                 | 约束/说明                          |
|--------------|------------------|----------------------|-----------------------------------|
| id           | bigint           | 主键 ID              | 自增，主键                        |
| book_isbn    | varchar(20)      | ISBN 编号            | 非空，唯一（图书唯一标识）        |
| book_name    | varchar(100)     | 图书名称             | 非空                              |
| author       | varchar(50)      | 作者                 | 非空                              |
| publisher    | varchar(50)      | 出版社               | 非空                              |
| publish_time | date             | 出版时间             | 非空                              |
| category_id  | bigint           | 分类 ID（关联分类表）| 非空，外键（关联 book_category.id） |
| price        | decimal(10,2)    | 图书定价             | 非空                              |
| cover_url    | varchar(255)     | 封面图片 URL         | 可选（可存储本地路径或OSS链接）   |
| description  | text             | 图书简介             | 可选                              |
| create_time  | datetime         | 创建时间             | 非空，默认 CURRENT_TIMESTAMP      |
| update_time  | datetime         | 更新时间             | 自动更新                          |

### 5. 图书库存表（book_stock）
单独存储库存，避免和图书基本信息耦合：
| 字段名       | 字段类型         | 注释                 | 约束/说明                          |
|--------------|------------------|----------------------|-----------------------------------|
| id           | bigint           | 主键 ID              | 自增，主键                        |
| book_id      | bigint           | 图书 ID（关联图书表）| 非空，外键（关联 book_info.id），唯一 |
| total_num    | int              | 总库存数量           | 非空，默认 0                      |
| borrow_num   | int              | 已借出数量           | 非空，默认 0                      |
| available_num| int              | 可借数量（计算字段） | 可选（也可通过 total_num - borrow_num 实时计算） |
| update_time  | datetime         | 更新时间             | 自动更新                          |

### 6. 图书借阅表（book_borrow）
存储借阅记录，核心业务表：
| 字段名       | 字段类型         | 注释                 | 约束/说明                          |
|--------------|------------------|----------------------|-----------------------------------|
| id           | bigint           | 主键 ID              | 自增，主键                        |
| book_id      | bigint           | 图书 ID              | 非空，外键                        |
| user_id      | bigint           | 借阅用户 ID          | 非空，外键                        |
| borrow_time  | datetime         | 借阅时间             | 非空，默认 CURRENT_TIMESTAMP      |
| due_time     | datetime         | 应归还时间           | 非空（如：借阅后 30 天）          |
| return_time  | datetime         | 实际归还时间         | 可选（未归还则为 null）           |
| status       | tinyint          | 状态（0-未还，1-已还，2-逾期） | 非空，默认 0                      |
| operator_id  | bigint           | 操作人 ID（管理员）  | 非空（借阅时的经办管理员）        |

### 7. 操作日志表（sys_log）
可选，用于审计和问题排查：
| 字段名       | 字段类型         | 注释                 | 约束/说明                          |
|--------------|------------------|----------------------|-----------------------------------|
| id           | bigint           | 主键 ID              | 自增，主键                        |
| user_id      | bigint           | 操作人 ID            | 非空                              |
| operation    | varchar(50)      | 操作类型             | 非空（如：借阅图书、新增图书）    |
| content      | varchar(255)     | 操作内容             | 可选（如：借阅图书《Java编程思想》） |
| ip           | varchar(20)      | 操作 IP              | 可选                              |
| create_time  | datetime         | 操作时间             | 非空，默认 CURRENT_TIMESTAMP      |

### 数据库设计补充
1. **索引**：为高频查询字段加索引（如 book_info.book_isbn、book_borrow.user_id、book_borrow.book_id），提升查询效率；
2. **外键约束**：关联表添加外键（如 book_borrow.book_id 关联 book_info.id），保证数据一致性；
3. **字符集**：所有表使用 `utf8mb4` 字符集，支持 emoji 和特殊字符。

## 三、核心功能模块与功能项
按“用户视角”和“模块维度”拆分，覆盖图书管理系统的核心场景：

### 1. 系统管理模块（管理员专属）
| 功能项               | 详细说明                                                                 |
|----------------------|--------------------------------------------------------------------------|
| 用户管理             | 新增/编辑/删除用户、重置密码、启用/禁用用户、查询用户列表（按角色/状态筛选） |
| 角色管理             | 新增/编辑/删除角色、为角色分配权限（如：管理员可操作图书，读者仅可借阅）|
| 操作日志查看         | 按时间/用户/操作类型查询日志、导出日志                                   |

### 2. 图书管理模块（管理员专属）
| 功能项               | 详细说明                                                                 |
|----------------------|--------------------------------------------------------------------------|
| 图书分类管理         | 新增/编辑/删除分类、分类排序、查询分类列表                               |
| 图书信息管理         | 新增图书（上传封面、填写基本信息）、编辑图书、删除图书、批量导入/导出图书 |
| 图书库存管理         | 调整库存数量、查看库存明细（总库存/已借出/可借）、库存预警（低于阈值提醒） |
| 图书查询             | 多条件查询（按名称/作者/ISBN/分类）、模糊搜索、分页展示                   |

### 3. 借阅管理模块（管理员+读者）
| 功能项               | 详细说明                                                                 |
|----------------------|--------------------------------------------------------------------------|
| 图书借阅（管理员）| 为读者办理借阅、选择图书+读者、自动计算应归还时间、更新库存             |
| 图书归还（管理员）| 办理归还、更新借阅状态、更新库存、标记逾期（如有）|
| 借阅记录查询（读者） | 查看自己的借阅记录、筛选（未还/已还/逾期）、查看应归还时间               |
| 借阅记录查询（管理员）| 按读者/图书/时间/状态查询所有借阅记录、导出记录、逾期提醒                 |

### 4. 前台门户模块（读者专属）
| 功能项               | 详细说明                                                                 |
|----------------------|--------------------------------------------------------------------------|
| 登录/注册            | 读者注册、账号密码登录、记住密码、忘记密码（邮箱/手机验证）|
| 图书浏览             | 首页展示热门图书、按分类浏览图书、图书详情页（查看简介/库存）|
| 个人中心             | 查看个人信息、修改密码、查看我的借阅记录                                 |


### 核心关键点回顾
1. **数据库设计**：核心表涵盖用户、角色、图书、库存、借阅五大类，字段设计兼顾业务和性能，通过外键保证数据一致性；
2. **功能模块**：分管理员和读者视角，覆盖“系统管理、图书管理、借阅管理、前台门户”四大核心模块，满足图书管理系统的核心需求；
3. **技术实现**：后端用 MyBatis-Plus 简化 CRUD，Spring Security 控制权限，事务保证数据一致性；前端用 Axios 封装请求，Element Plus 快速搭建UI，兼顾易用性和可维护性。
